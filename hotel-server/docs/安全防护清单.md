# 前后端安全防护清单

## 概述

本文档记录了酒店管理系统的安全防护措施，包括后端安全、前端安全、基础设施安全等方面。

**注意**: 本项目使用JWT Token进行身份认证，CSRF攻击风险较低，已禁用CSRF防护。

---

## 一、后端安全防护

### 1.1 XSS攻击防护 ✅
- **文件位置**: `hotel-web/src/main/java/com/xunni/hotel/web/filter/XssFilter.java`
- **实现方式**: 使用过滤器对所有请求参数进行HTML转义和脚本过滤
- **防护内容**:
  - HTML标签过滤
  - JavaScript脚本过滤
  - 事件处理函数过滤（onclick, onload等）
  - eval()函数过滤

### 1.2 CSRF攻击防护 ❌ 已禁用
- **说明**: 本项目使用JWT Token通过`Authorization: Bearer xxx`请求头传递，攻击者无法自动携带Token，因此CSRF攻击风险极低
- **替代方案**: JWT认证机制

### 1.3 点击劫持防护 ✅
- **文件位置**: `hotel-web/src/main/java/com/xunni/hotel/web/config/SecurityConfig.java`
- **实现方式**: 设置`X-Frame-Options: SAMEORIGIN`响应头
- **作用**: 防止页面被嵌入到iframe中

### 1.4 CORS跨域配置 ✅
- **文件位置**: `hotel-web/src/main/java/com/xunni/hotel/web/config/SecurityConfig.java`
- **配置**:
  - 允许所有来源（生产环境建议限制）
  - 暴露Authorization头
  - 支持凭证传递

### 1.5 敏感信息脱敏 ✅
- **文件位置**: 
  - `hotel-web/src/main/java/com/xunni/hotel/web/annotation/Sensitive.java`
  - `hotel-web/src/main/java/com/xunni/hotel/web/serializer/SensitiveJsonSerializer.java`
- **支持的脱敏类型**:
  - 手机号: `138****8888`
  - 邮箱: `1***@qq.com`
  - 身份证: `110****1234`
  - 银行卡: `6222****1234`
  - 姓名: `张**`
  - 地址: `北京市***区`

### 1.6 接口限流 ✅
- **文件位置**:
  - `hotel-web/src/main/java/com/xunni/hotel/web/annotation/RateLimit.java`
  - `hotel-web/src/main/java/com/xunni/hotel/web/aspect/RateLimitAspect.java`
- **限流类型**:
  - 按IP限流
  - 按用户限流
  - 按接口限流
- **默认配置**: 60秒内最多100次请求

### 1.7 安全日志记录 ✅
- **文件位置**: `hotel-web/src/main/java/com/xunni/hotel/web/interceptor/SecurityInterceptor.java`
- **记录内容**:
  - 请求ID
  - 客户端IP
  - 请求方法和URL
  - 请求时间
  - 响应状态码

### 1.8 全局异常处理 ✅
- **文件位置**: `hotel-web/src/main/java/com/xunni/hotel/web/config/GlobalExceptionHandler.java`
- **处理的异常类型**:
  - 认证异常 (401)
  - 权限拒绝 (403)
  - 参数校验失败 (400)
  - 服务器内部错误 (500)
  - SQL异常
  - 业务异常

### 1.9 密码策略 ✅
- **文件位置**: `hotel-web/src/main/java/com/xunni/hotel/web/config/PasswordPolicyConfig.java`
- **密码要求**:
  - 长度至少8位
  - 包含大写字母
  - 包含小写字母
  - 包含数字
  - 包含特殊字符 (@$!%*?&)

### 1.10 HSTS配置 ✅
- **文件位置**: `hotel-web/src/main/java/com/xunni/hotel/web/config/SecurityConfig.java`
- **配置**:
  - 包含子域名
  - 有效期1年 (31536000秒)

## 二、前端安全防护

### 2.1 Nginx安全配置 ✅
- **文件位置**: `nginx/conf.d/hotel.conf`
- **安全措施**:
  - 隐藏Nginx版本号
  - 添加安全响应头
  - 限制请求大小 (50MB)
  - 禁止访问敏感文件 (.git, .config等)
  - 禁止访问备份文件 (.bak, .old等)

### 2.2 静态资源缓存 ✅
- **配置**: 设置静态资源缓存时间为1年
- **资源类型**: js, css, png, jpg, gif, ico, svg

## 三、数据保护

### 3.1 密码加密存储 ✅
- **实现**: 使用BCrypt算法加密
- **文件位置**: `hotel-web/src/main/java/com/xunni/hotel/web/config/PasswordEncoderConfig.java`

### 3.2 JWT认证 ✅
- **实现**: 使用JWT Token进行无状态认证
- **文件位置**: `hotel-common/src/main/java/com/xunni/hotel/common/utils/JwtUtil.java`
- **优势**: 
  - 无状态认证，性能高
  - Token自带签名，防篡改
  - 通过请求头传递，防CSRF

### 3.3 SQL注入防护 ✅
- **实现**: 使用MyBatis参数化查询
- **防护原理**: 预编译语句防止SQL注入

## 六、安全事件响应流程

### 6.1 检测到异常
1. 安全日志记录异常请求
2. 系统自动记录攻击者IP

### 6.2 响应措施
1. IP临时封禁（通过限流机制）
2. 告警通知管理员
3. 记录安全事件日志

### 6.3 恢复流程
1. 分析攻击方式
2. 加固防护措施
3. 更新安全配置

## 七、安全最佳实践

| 安全原则 | 说明 |
|---------|------|
| **最小权限原则** | 用户只拥有完成工作所需的最小权限 |
| **纵深防御** | 多层防护措施，即使一层被突破仍有其他防护 |
| **数据脱敏** | 敏感数据在传输和存储时进行脱敏处理 |
| **定期审计** | 定期检查安全日志，发现异常行为 |
| **依赖更新** | 定期更新依赖，修复已知安全漏洞 |

## 附录：JWT与CSRF的关系

### 为什么JWT项目可以禁用CSRF？

| 对比项 | Cookie认证 | JWT认证 |
|-------|-----------|---------|
| **Token存储** | Cookie（自动发送） | 请求头（需手动设置） |
| **CSRF风险** | 高（Cookie自动发送） | 低（攻击者无法设置请求头） |
| **防护措施** | 需要CSRF Token | 通常不需要 |

### JWT认证流程

```
1. 用户登录 → 服务器返回JWT Token
2. 前端存储Token（localStorage/sessionStorage）
3. 后续请求通过 Authorization: Bearer <token> 发送
4. 服务器验证Token并处理请求
```

由于攻击者无法在第三方网站自动携带JWT Token（需要通过JavaScript设置请求头），因此CSRF攻击在JWT认证模式下几乎不可能成功。
